Getting Started with Markdown!

What is Markdown?

Markdown is a markup language that adds formatting elements to plaintext text documents. Basically, you use special syntax that makes your pages pretty. Essentially, you're writing HTML in simpler terms. When writing in Markdown, you can convert the ".md" file to HTML files! Markdown is useful because it can be used for many things (websites, emails, technical documentation), and a lot of platforms support it (Reddit, Github).

Useful Syntax

Here are some commonly used features of Markdown that are good to note!

Headings

Headings are denoted with the # symbol. A single # gives you a large heading (Heading 1), and more # symbols mean smaller headings (up to 6). Remember to leave a space between the # and your heading text! For heading levels 1 and 2, they can also be written by adding lines of = or - characters, respectively.

Paragraphs

Paragraphs don't require any special syntax. To insert a line break, add a couple spaces at the end of your sentence and hit return. Alternatively, you can use the HTML br tag if the spaces are too hard to keep track of.

Bold

By putting a word in between two asterisks or two underscores, it will make it bold. Haha, see what I did there?

Italic

By putting a word in between one asterisk or one underscore, voila! Italics.

Now Both!

I don't know why you would ever need something to be both bold and italic, but if you do, that calls for three asterisks.

Blockquotes

A > in front of a paragraph will make the text a blockquote. Insert a > with no text to make a new paragraph. You can even nest them by adding two of the arrows! You're also able to add other Markdown elements within the blockquote, but not all of them.

Lists

You can make both ordered and unordered lists. If you want to add a paragraph element or something other than a line item in between the list items, make sure to ident appropriately so it doesn't look wonky.

Ordered Lists

To make an ordered list, just add the items with numbers followed by periods. For example: "1. apples". The numbers don't have to be in numerical order, but they should start with the number one. You can indent for little inception lists.

  1. Did
  2. That
  3. Make
  4. Sense
    1. Hopefully?

Unordered Lists

To make an unordered list, dashes, asterisks, or plus signs in front of your items work. Same thing goes for the indentation as well.

  • Pineapple
  • Mango
  • Persimmon

Horizontal Rules

Make a horizontal break in your page using 3 or more asterisks, dashes, or underscores. Put a blank line before and after a horizontal rule just in case.

Links

Use brackets for the text you want displayed on the page, and parentheses for the actual URL. If you want a message to be displayed when the user is hovering on the link, just type in the message after the URL in quotations (while still in the parentheses). [Click here!](www.url.com "here's a message")
Angle brackets, <blah>, denotes a link as well. This method is good for emails!
Links can also be formatted with bold and italics.

Images

Add an exclamation point, followed by the alt text (what displays if the picture doesn't load), and then the path / URL. You can also add the title after the URL in quotations (while still in the parentheses).
Example: !Picture of Duck. []Picture of Duck.](https://images-na.ssl-images-amazon.com/images/I/71LgxJzwFKL.SL1500.jpg"duck")

Code

Using backticks around coding terms makes them appear in a special format! For example: pwd

Backslash

Using a backslash indicates you are using Markdown syntax, but you want it to show up regularly. (Symbols such as asterisks, underscores, etc).

HTML

If you forget the Markdown way to write it, have no fear. You can just use HTML tags in Markdown as well.

Converting to HTML

Since we've installed Pandoc, it's relatively easy to convert .md files to .html . Use the command "pandoc test.md -s -o test.html". We did it!
